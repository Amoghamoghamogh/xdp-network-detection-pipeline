# Architecture

## Overview

This project implements a kernel-space packet inspection and filtering pipeline
using XDP (eXpress Data Path) and eBPF. Packets are intercepted before SKB
allocation, enabling line-rate filtering with minimal kernel overhead.

## Pipeline

```
NIC (ingress)
     │
     ▼
┌─────────────────────────────────────────────────────┐
│  XDP Pipeline  (kernel/xdp_pipeline.bpf.c)          │
│                                                     │
│  1. Packet counter                                  │
│  2. Ethernet parse                                  │
│  3. IP parse          (IPv4 / IPv6)                 │
│  4. Transport parse   (TCP / UDP)                   │
│  5. IP firewall       LPM-trie ban check            │
│  6. Port filter       (IP, port) ban check          │
│  7. TCP fingerprint   JA4T-style SYN analysis       │
│  8. Latency tracking  JA4L-inspired RTT measurement │
└─────────────────────────────────────────────────────┘
     │                    │
  XDP_DROP             XDP_PASS
  (malicious)          (forwarded to kernel / Suricata)
```

## Module Breakdown

| File | Responsibility |
|------|---------------|
| `kernel/common.h` | Shared types, constants, inline packet-cursor helper |
| `kernel/maps.h` | All BPF map definitions and associated key/value structs |
| `kernel/modules/firewall.h` | IP ban (LPM trie) and port ban checks |
| `kernel/modules/tcp_fingerprint.h` | JA4T SYN fingerprinting and block-list enforcement |
| `kernel/modules/latency.h` | JA4L TCP handshake RTT tracking |
| `kernel/xdp_pipeline.bpf.c` | Pipeline entry point — wires all modules together |
| `userspace/loader.h` | libbpf loader stub (to be implemented) |

## BPF Maps

### IP Ban Maps
- `banned_ipv4` / `banned_ipv6` — LPM_TRIE, permanent bans (CIDR support)
- `recently_banned_ipv4` / `recently_banned_ipv6` — LPM_TRIE, temporary bans

### Port Ban Maps
- `banned_ports_inbound_v4/v6` — keyed by (src IP, src port)
- `banned_ports_outbound_v4/v6` — keyed by (dst IP, dst port)

### TCP Fingerprint Maps
- `tcp_fp_v4` / `tcp_fp_v6` — compound key (IP, port, fingerprint), full data
- `tcp_fp_simple_v4` / `tcp_fp_simple_v6` — simple (IP, port) key for O(1) userspace reads
- `blocked_tcp_fp_v4` / `blocked_tcp_fp_v6` — fingerprint block lists

### Latency Maps
- `conn_latency_v4` / `conn_latency_v6` — per-connection SYN/ACK/ACK timestamps

### Statistics Maps
- `stats_pkts_total`, `stats_pkts_dropped`
- `stats_ipv4_banned`, `stats_ipv6_banned`, etc.
- `dropped_ipv4_counters`, `dropped_ipv6_counters` — per-IP drop counts

## TCP Fingerprint Format (JA4T-style)

```
TTT:MMMM:WWWWW
│    │    └── TCP window size (5 digits)
│    └──────── MSS (4 digits)
└───────────── TTL / hop limit (3 digits)
```

14 bytes, ASCII, no null terminator. Used as a map key for O(1) block-list lookup.

## Latency Tracking (JA4L-inspired)

XDP only sees inbound packets. The approximation used:

- `syn_time_ns`    = timestamp of inbound SYN
- `synack_time_ns` = same as SYN (kernel responds near-instantly; SYN-ACK is outbound)
- `ack_time_ns`    = timestamp of inbound ACK

Client RTT ≈ `ack_time_ns - syn_time_ns`

## Userspace Control Plane (Future)

The userspace loader (`userspace/loader.h`) will use the libbpf skeleton
generated by `make` to:

1. Load and attach the XDP program
2. Poll BPF maps for telemetry
3. Push dynamic rule updates (ban IPs, block fingerprints)
4. Feed data to an IDS (e.g. Suricata) or SIEM

## Build

```bash
# Generate vmlinux.h from running kernel
make vmlinux

# Compile BPF object and generate libbpf skeleton
make

# Attach to interface (requires root)
sudo ./scripts/attach.sh eth0 native

# Detach
sudo ./scripts/detach.sh eth0
```
